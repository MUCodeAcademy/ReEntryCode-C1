# Projects

## Age calculator app

Your objective is to build out this age calculator app; look at ageApp.png for an example.

View an age in years, months, and days after submitting a valid date through the form.

Receive validation errors if:
- Any field is empty when the form is submitted
- The day number is not between 1-31
- The month number is not between 1-12
- The date is in the future
- The date is invalid, e.g. 31/04/1991 (there are 30 days in April)
View the optimal layout for the interface depending on their device's screen size
Set hover and focus states for all interactive elements on the page

## Matching game

Your objective is to build a 9x9 grid with items that have matching symbols.

- The items have symbols which are hidden at the beginning
- If the user clicks one, it displays the item
- If the user clicks another and it matches, those stay shown
- If the user clicks another and it does not match, hide both again
- Continue the game until the user matches all of them

## Hangman

Your objective is to build a game where the user guesses a word.

- Display the length of the chosen word
- The user can either use buttons on the screen, or their own keyboard to guess letters
- If the user guesses a letter and it's correct, put that letter in the word
- If the user guesses a letter and it's not in the word, give them that feedback

## Tic-Tac-Toe

Your objective is to build a simple web-based Tic-Tac-Toe game where a user can play against the computer.

- The game board is a 3x3 grid. 
- Each cell in the grid can be empty or contain an 'X' or an 'O'.
- The game starts with the human player as 'X' and the computer as 'O'.
- Players take turns placing their symbol on the board in an empty cell.
- The human player chooses a cell by clicking on it.
- The computer player chooses a cell based on some pre-defined logic. It could be random, or it could follow a strategy.
- A player wins the game if they get three of their symbols in a row, either horizontally, vertically, or diagonally.
- If all cells are filled and no player has won, the game is a draw.
- When the game ends, display a message showing who won or if it was a draw.

Bonus:
- Allow the user to choose whether they want to play as 'X' or 'O'.
- Implement a more sophisticated AI for the computer player. For example, it could try to win the game, or block the human player from winning.
- Allow the user to play again without refreshing the page.
- Keep track of and display the number of games won by the human player, the computer, and the number of draws.
